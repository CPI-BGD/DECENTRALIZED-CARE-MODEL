# Contributing to HCV-CARE Repository

Thank you for your interest in contributing to this comprehensive healthcare resource! This repository serves as the "Single Source of Truth" for the Decentralized HCV & CVD Care Model in Cox's Bazar.

---

## 🤝 How You Can Contribute

### **1. Report Issues or Suggest Improvements**
- Found an error or outdated information?
- Have a suggestion for improvement?
- Encountered a problem implementing the guidelines?

**How to report:**
1. Click the "Issues" tab on GitHub
2. Click "New Issue"
3. Describe the problem clearly
4. Provide context and any relevant details

### **2. Submit Updates or Corrections**
- Identified a typo or error?
- Want to update information based on your experience?
- Have a better way to explain something?

**How to submit:**
1. Fork the repository (click "Fork" button)
2. Make your changes
3. Submit a Pull Request with description of changes
4. Our team will review and merge

### **3. Translate Materials**
- Want to translate documents to Rohingya, Bangla, or other languages?
- Create translated versions in `/docs/translations/[language]/`

### **4. Add Local Adaptations**
- Implemented this model in your area?
- Have adaptations specific to your context?
- Add case studies in `/docs/07-CASE_STUDIES/`

### **5. Share Implementation Experience**
- Share successes and lessons learned
- Contribute to case studies
- Provide feedback on what worked/didn't work

### **6. Develop Tools & Resources**
- Created Excel calculators or data tools?
- Developed training videos or infographics?
- Built mobile app or database?
- Submit for inclusion with proper attribution

## 📋 Guidelines for Contributing

### **Before You Start**
- [ ] Review existing content (avoid duplication)
- [ ] Check open Issues and Pull Requests
- [ ] Ensure your contribution aligns with program goals
- [ ] Verify information is evidence-based

### **Content Standards**
1. **Accuracy**: All information must be evidence-based and factually correct
2. **Clarity**: Use simple language; avoid jargon without explanation
3. **Completeness**: Provide context and full information
4. **Structure**: Follow existing format and structure
5. **Attribution**: Credit sources and authors

### **Writing Guidelines**
- **Tone**: Professional, respectful, patient-centered
- **Language**: Simple, clear, direct
- **Length**: Comprehensive but not verbose
- **Formatting**: Use headings, bullets, tables for readability
- **Language Level**: Appropriate for varied literacy levels

### **File Naming Conventions**
```
docs/
├── 01-STRATEGIC_FRAMEWORK/
│   ├── 01_Program_Overview.md
│   ├── 02_Evidence_Base.md
│   └── 03_Target_Populations.md
├── 02-IMPLEMENTATION_GUIDES/
│   ├── 01_Organizational_Structure.md
│   ├── 02_Implementation_Roadmap.md
│   └── 03_Resource_Planning.md
```

- Use **numbers** for ordering (01_, 02_, etc.)
- Use **underscores** between words (not spaces or hyphens)
- Use **descriptive names** that indicate content
- Use **.md** extension (Markdown format)

### **Markdown Format Standards**
```markdown
# Title (H1 - use once per page)
## Section (H2)
### Subsection (H3)
#### Detail (H4)

- Use bullet points
- For lists
- Keep readable

**Bold** for emphasis
*Italic* for secondary emphasis
[Link text](url) for hyperlinks

| Column 1 | Column 2 |
|----------|----------|
| Data     | Data     |

> Block quotes for
> important information
```

### **Image Guidelines**
- Store in `/assets/images/`
- Use descriptive filenames: `patient_journey_hcv.png`
- Include alt text: `![Patient journey flowchart](../assets/images/patient_journey_hcv.png)`
- Keep file sizes <500KB
- Use PNG or JPG format

### **Code & Data Files**
- Store scripts in `/tools/`
- Include README with instructions
- Provide example data files
- Document how to use (step-by-step)

## 🚀 Submission Process (Step-by-Step)

### **Option 1: GitHub Web Interface (Easiest)**

**To fix a small typo or suggest text:**
1. Navigate to the file you want to edit
2. Click the pencil icon (✏️) in top right
3. Make your changes
4. Click "Propose Changes"
5. Add description: "What changed and why"
6. Click "Create Pull Request"
7. Wait for review

**To add a new file:**
1. Navigate to the folder where you want to add a file
2. Click "Add file" → "Create new file"
3. Name the file (use convention: `01_File_Name.md`)
4. Add content
5. Click "Propose new file"
6. Create Pull Request

### **Option 2: GitHub Desktop (For Non-Coders)**

1. Install [GitHub Desktop](https://desktop.github.com/)
2. Sign in with your GitHub account
3. Click "Clone a Repository"
4. Search for `CPI-BGD/HCV-CARE`
5. Clone to your computer
6. Make changes in your preferred editor
7. Switch to GitHub Desktop
8. Changes appear in "Changes" tab
9. Write summary of changes
10. Click "Commit to main"
11. Click "Publish branch"
12. On GitHub website, Create Pull Request

### **Option 3: Collaborative Editing (Most Accessible)**

1. Contact [healthbgd@cpintl.org](mailto:healthbgd@cpintl.org)
2. Request shared Google Doc link for collaboration
3. Make edits in Google Doc
4. Team integrates into GitHub

## 🔄 Review Process

**Timeline:**
1. **Submission**: You submit Pull Request
2. **Acknowledgment** (24 hours): Team reviews and provides initial feedback
3. **Review** (2-7 days): Detailed content review; may request changes
4. **Revision** (as needed): You make requested changes
5. **Approval** (1-3 days): Final approval and merge
6. **Publication** (same day): Changes appear on website

**Types of feedback:**
- ✅ **Approved**: Will be merged as-is
- 📝 **Changes requested**: Please revise and resubmit
- ❓ **Questions**: Team will ask for clarification
- 💬 **Comments**: General discussion or suggestions

## 🌐 Translation Contributions

Want to translate this to other languages?

1. Choose language
2. Create folder: `/docs/translations/[language-code]/`
   - Example: `/docs/translations/bn/` for Bengali
   - Example: `/docs/translations/ro/` for Rohingya
3. Translate core documents in order:
   - Program Overview
   - Implementation Guide
   - Training Modules
   - SOPs
4. Submit Pull Request with translation

**Supported languages (currently):**
- 🇧🇩 Bengali (partial)
- 🇷🇴 Rohingya (needed)
- 🇲🇲 Burmese (needed)
- 🇫🇷 French (for international use)
- 🇪🇸 Spanish (for international use)

**Translation guidelines:**
- Maintain original meaning and intent
- Adapt examples to local context if helpful
- Use professional medical terminology
- Include translator name and date
- Mark as "Beta/Draft" until reviewed by native speaker
